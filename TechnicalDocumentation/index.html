Introduction

ABAP stands for Advanced Business Application Programming. It is a programming language used to develop and customize SAP.

It was created with the intention to be used by SAP clients to customize their products, but it got so sophisticated that requires some advanced programming knowledge to best take advantage of it. Therefore, it has been used for programmers to enhance and customize SAP for the customer.

This tutorial doesn't expect that you have profound knowledge of programming but assumes you understand some technical terms

How to start

To program in ABAP, one can use SAP GUI or ADT. SAP GUI comes with the installation of SAP NETWEAVER while ADT can be installed without SAP NETWEAVER installed.


With SAP GUI, one can create its first program by following the steps bellow:

  1. Go to transaction SE80
  2. On the left panel choose program
  3. Bellow it insert the name of the program, ex. ZPR_HELLO_WORLD
  4. Click on enter
  5. It would show a pop-up saying the program doesn't exist and ask if you want to create it. Click on "Yes"
  6. On the new pop-up just keep clicking on "Yes" until it gets to a pop-up requesting a package
  7. On package pop-up select "local object"and done, your first program is created
  8. Double click on the program and you would be able to see the code of your report
  9. To change between edit and visualize, use the shortcut ctrl+F1

With SAP ADT, one can create its first program by following the steps bellow:

  1. use the shortcut ctrl+shift+N
  2. Type program and select ABAP Program
  3. On next screen fill in those values:
    • Package: $TMP
    • Name: ZPR_HELLO_WORLD
    • Description: My firs hello world program
  4. Click on finish
  5. Your program is created, in eclipse, you don't need to change between edit and see. When you start typing, eclipse changes the status from display to edit automatically

Now that we have our first program created, let's display "hello world" to anyone who runs this program by following the steps bellow:

  1. Open the program if closed. For SAP GUI, follow the steps 1 to 4 on how to create a program. If SAP ADT (eclipse) use the shortcut ctrl+shift+A, insert the name of the program e select it to open.
  2. Type the code write 'Hello world'.with the dot at the end
  3. Activate the program using the shortcut ctrl+F3
  4. Run the program by using the key F8
Your code would look like the following:

  1. report ZPR_HELLO_WORLD.
  3. write 'Hello world'.

Congrats, your first ABAP program works.

Note: All ABAP objects created by a customer should start with the letter Z or Y

ABAP statement

All statements in ABAP are finished by a dot (.). If you finish a code line without putting a dot at the end, ABAP will raise error and fail to activate.

Many statements start with a special ABAP keyword such as data, write, call function, etc.

If you would start many lines with the same keyword, you can use colon and commas to reduce the number of repetition. For example, let's assume you want to create three variables first_variable, second_variable and third_variable. In ABAP you could code like one of the following:

  1. report ZPR_HELLO_WORLD.
  3. data: gv_first_variable.
  4. data: gv_second_variable.
  5. data: gv_third_variable.

OR

  1. report ZPR_HELLO_WORLD.
  3. data: gv_first_variable,
  4. gv_second_variable,
  5. gv_third_variable.

On first example, there were three statements starting with the keyword data and finishing with a dot, whereas on second example, it was used the word data only once followed by colon (:) that allows to combine multiple statements in one separating them with a commas and placing a single dot at the very end as can be see above.

Note: the two codes have exactly the same effect and result

ABAP Syntax

ABAP words and statements are case insensitive, that means that the following lines of code would lead to exactly same result:

  1. write: 'Hello ', gv_full_name.
  2. WRITE: 'Hello ', GV_FULL_NAME.
  3. Write: 'Hello ', GV_Full_Name.

The three lines above would write to the screen "Hello " followed by the value in variable full_name

Elementar Data Types

The table bellow shows the elementar data types available in SAP ABAP

Data Type Description
C Character string, default length of 1 but can be changed
N Numerical character, default length of 1 but an be changed. Accepts only numerical value but cannot do math operation
I Integer, accepts negative value
P Packed number, length and decimals should be determined
F Floating point number, length and decimals should be determined
DECFLOAT16, DECFLOAT32 Decimal Floating point, kind of combination of advantage from P and F, those types have fixed length and decimals
D Date, format YYYYMMDD
T Time, format HHMMSS
X Hexadecimal or byte sequence,
No Elementar Data Types

As ABAP is created on a business environment, there are thousands or even millions of available data type. Actually, any table created in database can be used to create a local structure or table in an ABAP program. Nevertheless, some most used data types are:

  • string: similar to C but with dynamic length
  • xstring: similar to X but with dynamic length
  • abap_bool: for boolean values, false is empty and true is letter "x"
Variables

ABAP is case insensitive, therefore, you cannot use upper/lower case to diferentiate variables as already shown above.

Most ABAP variables actually fall unto one of the following kinds:

  • Variable → Any variable that would accept any elementar data type
  • Structure → Variable that would accept any combination of elementar data types, usually accepts 2 or more values, ie., first and last name
  • Table → Basically two or more lines of structure combined on a single variable
  • Object → Contains reference to an object, such as a class

Note: You must declare a variable before use it. Since SAP NW7.4, you can declare a variable when you need to use it, that is, inline declaration.

Declaring a Variable

The convectional naming of a variable in SAP consist in the following:

  • The first letter should say the scope of the variable, as a result, it should be G for global variable or L for local variable
  • The second letter should say the kind of variable with the following possibilities:
    • V for variable
    • S for Structure
    • T for Table
    • O for Reference or Object or Class
  • The third character should be an underscore, and then, the name of the variable

An example would be ls_student, by analysing the first 2 letters and the name of it, I would conclude that it's a structure that is accessible only within this block and may be filled with data related to a student such as first name, last name, student id, etc.

Even without seeing the data declaration, I can infer what is in it and its scope.


To declare a variable, we use the keyword data, see examples bellow with comments

  1. data: gv_variable1,
  2. gv_variable2 type c,
  3. gv_variable3 type c length 1.
  4. * The three variables above accepts only 1 character,
  5. * they're equivalent
